## 1. 文档概述

### 1.1 背景

当前AI绘图工具面临的主要挑战是：用户需要具备专业的提示词工程知识才能获得理想的生成结果。普通用户难以准确描述摄影参数、光影效果、艺术风格等专业要素。

### 1.2 目标

通过引入多智能体协作架构，构建一个**意图理解→参数匹配→提示词生成**的自动化工作流，降低用户使用门槛，同时通过人机协同机制保留专业用户的掌控权。

### 1.3 适用范围

本PRD适用于Cloudrawn彩云间Chrome扩展的智能增强功能模块开发。

---

## 2. 产品目标

### 2.1 用户价值

1. **降低门槛**：用户只需自然语言描述想法，系统自动转换为专业提示词
2. **提升质量**：自动匹配摄影、美术、建筑等专业参数，提升生成质量
3. **保持掌控**：用户可审阅和调整AI推荐的参数，实现人机协同

### 2.2 业务目标

- 提升用户生成满意度（目标：提示词接受率 > 80%）
- 减少用户提示词编辑时间（目标：平均缩短 60%）
- 支持多种专业场景（建筑、室内、产品、艺术等）

## 3. 用户故事

### 故事1：建筑设计师
> 作为建筑设计师，我希望输入"现代风格博物馆，黄昏时分"后，系统自动为我匹配专业的建筑摄影参数（如 tilt-shift 镜头、f/11 光圈），让我快速获得专业级效果图。

### 故事2：城市规划师
> 作为城市规划师，我需要生成城市更新效果图。我希望系统能自动识别"宏大场景"意图，并匹配广角镜头、鸟瞰视角等参数，无需我手动输入专业术语。

### 故事3：普通用户
> 作为普通用户，我想画一张"雨后的极简风格美术馆"。我希望系统理解我的意图后，自动推荐合适的光影（柔和散射光）、材质（混凝土纹理）等参数，并且可以让我选择是否接受。

## 4. 系统架构

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      用户交互层                               │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │ 自然语言输入框 │  │ 参数控制台    │  │ 生成结果预览  │       │
│  └──────────────┘  └──────────────┘  └──────────────┘       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    Agent工作流引擎                           │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐  │
│  │ 节点A：意图解析 │───▶│ 节点B：参数映射 │───▶│ 节点C：提示词 │  │
│  │ Intent Analyzer│    │  RAG检索增强  │    │   融合输出   │  │
│  └──────────────┘    └──────────────┘    └──────────────┘  │
│         │                    │                    │         │
│         ▼                    ▼                    ▼         │
│    提取结构化标签      匹配专业参数库         生成英文Prompt │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 三节点工作流设计

#### 节点A：意图解析 Agent (Intent Analyzer)

**输入**：用户的原始自然语言输入

**输出**：JSON格式的结构化标签

```json
{
  "subject": "现代风格博物馆",
  "time_lighting": "黄昏",
  "mood_style": "赛博朋克",
  "scale": "城市更新/宏大场景",
  "additional_notes": ["雨后", "极简风格"]
}
```

**系统提示词核心逻辑**：
```
你是一个意图分析引擎。请阅读用户的输入，并输出 JSON 格式的解析结果，
包含四个维度：
1. 核心主体 (Subject)
2. 时间/光照 (Time/Lighting)
3. 氛围/风格 (Mood/Style)
4. 视角大小 (Scale)
```

#### 节点B：参数映射与增强 Agent (Parameter Mapping & RAG)

**输入**：节点A输出的结构化标签

**任务**：充当"摄影指导"和"美术指导"，自动匹配专业参数

**参数匹配规则库**：

| 意图维度 | 自动挂载参数 |
|---------|-------------|
| Scale=城市更新/宏大场景 | 14mm ultra-wide angle, f/11 aperture, bird's-eye view |
| Time/Lighting=傍晚 | Blue Hour, glowing neon lights, cinematic lighting |
| Mood/Style=赛博朋克 | Cinestill 800T film, red halation, high contrast |
| Subject=建筑/城市景观 | 24mm tilt-shift lens, f/11, architectural photography |
| Subject=人物/特写 | 85mm lens, f/1.4, shallow depth of field |
| Mood=电影感 | ARRI Alexa 65, cinematic lighting, volumetric light |
| Style=真实感 | Kodak Portra 400, 8k resolution, hyper-realistic |

**输出**：增强后的参数标签列表

```json
{
  "lens": ["24mm tilt-shift", "f/11"],
  "lighting": ["Blue Hour", "cinematic lighting"],
  "film": ["Cinestill 800T"],
  "effects": ["red halation", "high contrast"],
  "quality": ["8k resolution", "architectural photography"]
}
```

#### 节点C：提示词融合与输出 Agent (Prompt Synthesizer)

**输入**：用户原始需求 + 节点B匹配的专业参数

**输出**：符合AI绘图模型语法的英文提示词

**输出格式**：
```
[核心主体细节], [环境背景], [光影与天气], [匹配的摄影镜头参数], [匹配的胶片/渲染引擎参数], [画质修饰词]
```

**示例输出**：
```
A minimalist museum building with concrete texture, after rain with puddle reflections, 
overcast soft diffused light, 24mm tilt-shift lens, f/8 aperture, 
shot on Kodak Portra 400, 8k resolution, hyper-realistic architectural photography
```

## 5. 人机协同交互设计

### 5.1 核心原则

**非黑盒生成**：虽然Agent在后台完成复杂匹配，但在产品交互上保留"确认与微调"环节，让专业用户保持掌控感。

### 5.2 推荐交互流程

```
┌────────────────────────────────────────────────────────────┐
│ Step 1: 用户输入                                            │
│ ┌────────────────────────────────────────────────────────┐ │
│ │ 一个雨后的极简风格美术馆，黄昏时分                        │ │
│ └────────────────────────────────────────────────────────┘ │
│                              │                             │
│                              ▼                             │
│ Step 2: 点击"智能增强"按钮                                   │
│ ┌────────────────────────────────────────────────────────┐ │
│ │              🚀 智能增强                                  │ │
│ └────────────────────────────────────────────────────────┘ │
│                              │                             │
│                              ▼ (1-2秒处理)                  │
│ Step 3: 参数控制台弹出 (Spotlight/侧边栏)                    │
│ ┌────────────────────────────────────────────────────────┐ │
│ │  📋 Agent为您匹配了以下参数：                              │ │
│ │                                                          │ │
│ │  镜头    [24mm Tilt-Shift] [f/8]              [+] [-]   │ │
│ │  光照    [Blue Hour] [Soft Diffused Light]    [+] [-]   │ │
│ │  材质    [Concrete texture] [Puddle reflections] [+] [-]│ │
│ │  胶片    [Kodak Portra 400]                   [+] [-]   │ │
│ │                                                          │ │
│ │  [✓ 确认生成]  [🔄 重新分析]  [✏ 手动编辑]              │ │
│ └────────────────────────────────────────────────────────┘ │
│                              │                             │
│                              ▼                             │
│ Step 4: 用户确认后生成图像                                    │
└────────────────────────────────────────────────────────────┘
```

### 5.3 参数控制台功能

| 功能 | 描述 |
|-----|------|
| Tag展示 | 以可交互标签形式展示Agent推荐的参数 |
| 删除Tag | 点击"X"删除不想要的参数 |
| 添加Tag | 手动输入或从预设列表选择新参数 |
| 编辑模式 | 切换到文本编辑模式直接修改提示词 |
| 重新分析 | 重新触发Agent工作流 |
| 确认生成 | 使用当前参数生成图像 |

## 6. 技术实现

### 6.1 技术选型

| 组件 | 推荐方案 |
|-----|---------|
| LLM模型 | Qwen-plus / Claude / GPT-4 |
| Agent框架 | 单Agent多节点（CoT思维链）或多Agent协作 |
| 参数知识库 | 本地JSON文件 + 向量检索（可选） |
| UI组件 | 原生JavaScript + CSS |

### 6.2 核心Prompt模板

```
你是一个专业的 AI 绘图提示词构建 Agent。
你的任务是：读取用户的简短输入，分析其意图，并自动匹配顶级的摄影参数和艺术风格，
最终输出一段结构化的英文 Prompt。

【参数匹配规则库】
1. 建筑/城市景观意图：强制使用 "24mm tilt-shift lens", "f/11 aperture", "architectural photography"。
2. 人物/特写意图：强制使用 "85mm lens", "f/1.4 aperture", "shallow depth of field"。
3. 电影感/情绪意图：匹配 "shot on ARRI Alexa 65", "cinematic lighting", "volumetric light"。
4. 真实感/写实意图：匹配 "Kodak Portra 400" 或 "Fujifilm Velvia 50", "8k resolution", "hyper-realistic"。

【工作流要求】
1. 分析用户输入的主体和场景。
2. 从上述规则库中选取最匹配的 1-2 组参数。
3. 使用英文输出最终的绘图 Prompt，遵循格式：
   [核心主体细节], [环境背景], [光影与天气], [匹配的摄影镜头参数], [匹配的胶片/渲染引擎参数], [画质修饰词]

用户输入：{user_input}
请先简要用中文输出你的【分析过程】，然后输出最终的【英文 Prompt】。
```

### 6.3 集成方案

```javascript
// 智能增强按钮点击事件
async function handleSmartEnhance() {
    const userInput = getUserInput();
    
    // 显示加载状态
    showLoading();
    
    try {
        // 调用 Agent API
        const result = await callAgentAPI(userInput);
        
        // 解析返回结果
        const { analysis, prompt, tags } = parseAgentResult(result);
        
        // 显示参数控制台
        showParameterConsole({
            analysis,
            prompt,
            tags
        });
    } catch (error) {
        showError(error);
    }
}
```

## 7. 验收标准

### 7.1 功能验收

| 验收项 | 验收标准 |
|-------|---------|
| 意图解析准确率 | 能够正确提取Subject/Time/Mood/Scale四个维度，准确率>90% |
| 参数匹配覆盖率 | 支持建筑、人物、风景、产品等至少5种场景的参数匹配 |
| 提示词质量 | 生成的英文提示词语法正确，参数搭配合理 |
| 人机协同流程 | 完整的输入→分析→确认→生成流程可用 |

### 7.2 性能验收

| 指标 | 目标值 |
|-----|-------|
| 单次Agent处理时间 | < 3秒 |
| UI响应时间 | < 100ms |
| 参数控制台渲染 | < 200ms |

### 7.3 用户体验验收

| 场景 | 预期结果 |
|-----|---------|
| 新用户首次使用 | 能理解"智能增强"功能的作用和使用方法 |
| 专业用户调整参数 | 能快速删除/添加Tag，编辑流畅 |
| 错误输入处理 | 对模糊输入给出合理推测，并提示用户确认 |

## 8. 风险与依赖

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| LLM响应不稳定 | 用户体验差 | 添加超时重试机制，提供降级方案（本地规则匹配） |
| 参数匹配不准确 | 生成效果差 | 建立用户反馈机制，持续优化规则库 |
| 处理时间过长 | 用户流失 | 添加进度提示，考虑流式输出 |

### 8.2 外部依赖

| 依赖 | 说明 |
|-----|------|
| LLM API | 需要稳定的LLM服务（阿里云百炼Qwen或其他） |
| 浏览器兼容性 | 确保在主流Chrome版本正常运行 |
